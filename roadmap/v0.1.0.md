## Critical path (must all be DONE to reach v0.1.0)

### CP-1 — Real-browser end-to-end authoring path
**Status: BLOCKED on evidence (probe commissioned).**
Every e2e proof in the repo terminates at jsdom; the POC integration test explicitly flags
"runs in a real browser, interaction updates the DOM" as an unmet handoff. `mount`'s `$style`
scope-attr stamping needs `root instanceof Element`, undefined under inline-jsdom.

- **CP-1a — `.nv` → esbuild plugin → bundle → mount → interact, in a real browser.** The
  `nvPlugin()` esbuild loader exists (`.nv` → `parseNvFileForEmit` → `emitModule` → JS) but has
  **no end-to-end build test**. Required: a real Vite/esbuild build of a small `.nv` app, loaded in
  a real page, Playwright-asserted that a click updates the DOM. **NOT STARTED.**
- **CP-1b — event-handler → signal-write authoring works in `.nv`.** IR has `event` binding;
  unverified whether authoring `onClick={() => count.set(...)}` in `.nv` parses and wires
  end-to-end. **PROBE COMMISSIONED** → `handoff-nv-author-probe-SONNET.md`. Result feeds this
  checkpoint: "documented pattern" vs "missing wiring."

### CP-2 — js-framework-benchmark keyed implementation
**Status: NOT STARTED (one OPEN QUESTION inside).**
A `frameworks/keyed/nv/` implementation that builds via `npm run build-prod` and passes the
webdriver checks.

- **CP-2a — author the benchmark app in `.nv`** (table, action-trigger buttons with exact ids,
  Bootstrap CSS, row select/swap/remove). Exercises `list` (keyed), `event`, `classlist`
  (selected-row toggle). **NOT STARTED.**
- **CP-2b — keyed-swap compliance (VERIFY, reopen NOT expected).** The benchmark's keyed-swap rule
  requires *moving* the actual DOM rows, not remove+reinsert (else DOM state is lost). **Source read
  (`interpreter.ts` L534–544): the reconcile already moves existing `rootEl` nodes via
  `insertBefore` in reverse** — and `insertBefore` on a node already in the document *moves* it
  (preserves element identity and DOM state). The 2026-06-22 LIS closure also records single-move as
  O(1) DOM mutations. So nv **likely already satisfies `isKeyed` for swap by construction.** Action
  is to **verify** `npm run isKeyed keyed/nv` passes once CP-2a exists — *not* to reopen LIS. LIS
  reopen would only be triggered by a *measured* swap-throughput deficit at benchmark scale, a
  separate and currently-unsupported hypothesis. **Do not pre-commission an LIS reopen.** → LIS
  closure: decision-log 2026-06-22.
- **CP-2c — record baseline numbers** vs Solid/Svelte/vanilla on the standard ops (create/replace/
  update/select/swap/remove/create-10k/append/clear + memory). Real-hardware, CC-run. **NOT
  STARTED.**

### CP-3 — Public API is authorable without IR knowledge
**Status: OPEN QUESTION.**
Today `mount(ir, parent, doc)` takes an IR. App authors write `.nv` or tagged-template, never raw
IR — but confirm there is a clean documented "here is how you start an app" entry (the `$component`/
`$render` authoring surface → bundle → mount). If the only mount entry is IR-level, v0.1.0 needs a
thin author-facing `mount(Component)` or documented build-output convention. **Probe (CP-1b) informs
this.**

### CP-4 — Minimal documentation for the above
**Status: NOT STARTED.**
Enough to let someone author a `.nv` app: primitives, the four authoring macros (`$component`,
`$render`, `$style`, props), control flow (`<each>`, conditional), events, `sync`. Not full docs —
the floor that makes the benchmark app and a small clone writable by someone who isn't the architect.

## Already DONE (the engine floor — verified)

- **Reactive core** — `signal`, `derived`, `effect`, `sync`, `pubsub`, `errorBoundary`, `batch`,
  `untrack`, `createRoot`, `onCleanup`, `flushSync`. DOM-free, confirmed 2026-06-24. Contract v0.4.2.
- **IR binding vocabulary** — text, attr, prop, event, child, conditional, list (keyed), sync,
  component, slot-outlet, classlist, style-var. Sufficient for a HN/Twitter-class app's rendering.
- **Both front-ends** — `.nv` parser + tagged-template, FE-equivalence-gated, one shared IR.
- **Both back-ends** — interpreter + compiler/emitter (`emitMount`), differential-gated.
- **esbuild plugin** — `.nv` loader exists (untested e2e — see CP-1a).
- **Slot domain** — CLOSED `58afe25`; `<each>`-in-slot both FEs, scoped slots, `$style` scoping.
- **Wide-graph propagation** — EVIDENCE-TESTED CLEAR `387990f`; ~0.15–0.17ms/tick, ~100× under
  frame budget. Propagation is not a top user-facing cost at realistic scale.
